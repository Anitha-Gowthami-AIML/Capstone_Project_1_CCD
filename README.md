## 🧠 Overview

CreditLens solves the **credit card default classification** problem on the UCI dataset of 30,000 Taiwanese bank clients. The core challenge is **class imbalance** — only ~22% of clients default — which causes naive classifiers to under-predict the minority class.

The platform addresses this through three experimental groups:

| Group | Strategy | Threshold |
|-------|----------|-----------|
| **Baseline** | Standard training, no imbalance handling | 0.5 |
| **Imbalance-Handled** | SMOTE oversampling + Class weights | 0.3 |
| **Tuned** | RandomizedSearchCV hyperparameter search | 0.3 |

All 13 trained models are served through a Streamlit web app with real-time prediction, feature importance charts, ROC/PR curves, and confusion matrices.

## 📊 Dataset

| Property | Value |
|----------|-------|
| **Source** | UCI Machine Learning Repository |
| **URL** | https://archive.ics.uci.edu/ml/datasets/default+of+credit+card+clients |
| **Paper** | Yeh & Lien (2009), Expert Systems with Applications |
| **Records** | 30,000 credit card clients |
| **Period** | April – September 2005, Taiwan |
| **Target** | `DEFAULT` — 0 = No Default, 1 = Default |
| **Class split** | ~78% Non-Default / ~22% Default |

**Raw features:** 23 original columns covering credit limit, demographics (age, sex, education, marriage), 6 months of payment status, bill amounts, and payment amounts.

## ⚙️ Feature Engineering

`model_train.py` derives 9 additional features before training:

| Feature | Formula | Captures |
|---------|---------|---------|
| `UTIL_RATE` | `BILL_AMT1 / LIMIT_BAL` clipped [-1, 2] | Credit utilisation pressure |
| `AVG_PAY_STATUS` | `mean(PAY_0..PAY_6)` | Average payment delay severity |
| `TOTAL_BILL` | `sum(BILL_AMT1..BILL_AMT6)` | 6-month billing exposure |
| `TOTAL_PAY` | `sum(PAY_AMT1..PAY_AMT6)` | 6-month payment made |
| `PAY_RATIO` | `TOTAL_PAY / TOTAL_BILL` clipped [-2, 5] | Repayment capacity |
| `BILL_TREND` | `BILL_AMT1 - BILL_AMT6` | Direction of debt growth |
| `PAY_TREND` | `PAY_AMT1 - PAY_AMT6` | Direction of payment behaviour |
| `MAX_PAY_DELAY` | `max(PAY_0..PAY_6)` | Worst single-month delay |
| `CONSEC_LATE` | `count(PAY_x > 0)` | Months with any delay |

**Total training features: 32** (23 original + 9 engineered)

Two preprocessors are applied inside sklearn Pipelines:
- **Linear preprocessor** (Logistic Regression): Yeo-Johnson power transform on bill amounts, `log1p` on payment amounts, `StandardScaler` on numeric features, `OneHotEncoder` on categoricals
- **Tree preprocessor** (all tree models): `OneHotEncoder` on categoricals only — trees are scale-invariant

## 🤖 Models Trained

### Group 1 — Baseline (threshold = 0.5)

| Model | Key Settings |
|-------|-------------|
| Logistic Regression | `max_iter=1000` |
| Decision Tree | `max_depth=6` |
| Random Forest | `n_estimators=200` |
| Gradient Boosting | default params |
| XGBoost | `n_estimators=300, lr=0.05, max_depth=5` |

### Group 2 — Imbalance-Handled (threshold = 0.3)

| Model | Technique | Key Setting |
|-------|-----------|-------------|
| SMOTE Logistic Regression | SMOTE | `sampling_strategy=0.5` |
| SMOTE Random Forest | SMOTE | `sampling_strategy=0.5` |
| SMOTE XGBoost | SMOTE | `sampling_strategy=0.5` |
| Weight Logistic Regression | `class_weight='balanced'` | — |
| Weight Random Forest | `class_weight='balanced'` | `n_estimators=200` |
| Weight XGBoost | `scale_pos_weight` | ~3.52 (neg/pos ratio) |

### Group 3 — Tuned (threshold = 0.3)

`RandomizedSearchCV` with **20 iterations**, **5-fold StratifiedKFold**, `scoring='f1'`:

| Model | Search Space |
|-------|-------------|
| Tuned SMOTE XGBoost | `n_estimators`, `max_depth`, `learning_rate`, `subsample`, `colsample_bytree`, `sampling_strategy` |
| Tuned Weight XGBoost | `n_estimators`, `max_depth`, `learning_rate`, `subsample`, `colsample_bytree`, `gamma` |

## ⚡ Quickstart

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/creditlens.git
cd creditlens

# If models/ contains large .pkl files tracked by Git LFS:
git lfs pull
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Train the models

> ⚠️ **Run this before launching the app.** This generates the `models/` directory with all `.pkl` files and summary JSON/CSV files.

```bash
python model_train.py
```

## 🗃️ Git LFS Setup (for large model files)

Random Forest `.pkl` files can exceed **150–400 MB** each — well above GitHub's 100 MB file limit. Git LFS is required to push them.

### One-time setup

```bash
# Install Git LFS (once per machine)
git lfs install

# Track all pickle files
git lfs track "*.pkl"
git lfs track "models/*.pkl"

# IMPORTANT: commit .gitattributes BEFORE staging any .pkl files
git add .gitattributes
git commit -m "chore: configure Git LFS for model pkl files"
```

### Push workflow

```bash
git add app.py model_train.py requirements.txt runtime.txt README.md
git add Credit_Card_Default.csv
git add models/

# Verify LFS will handle the large files
git lfs status

git commit -m "feat: add CreditLens app, trained models, and evaluation artifacts"

git remote add origin https://github.com/<username>/creditlens.git
git push -u origin main
# LFS uploads large files separately — you'll see progress for each .pkl
```

### Cloning on a new machine

```bash
git clone https://github.com/<username>/creditlens.git
cd creditlens
git lfs pull          # downloads actual .pkl binaries
pip install -r requirements.txt
python model_train.py # OR skip if lfs pull brought the pre-trained models
streamlit run app.py
```

> **GitHub Free plan** includes 1 GB LFS storage and 1 GB bandwidth/month. If you hit limits, consider only storing the two tuned models (`tuned_smote_xgb.pkl`, `tuned_weight_xgb.pkl`) in LFS and regenerating the rest via `model_train.py`.

## 📈 Results

Expected benchmark performance on the 20% held-out test set:

| Model | AUC-ROC | F1 | Recall | Precision |
|-------|---------|-----|--------|-----------|
| Baseline XGBoost | ~0.78 | ~0.47 | ~0.37 | ~0.64 |
| SMOTE XGBoost | ~0.77 | ~0.52 | ~0.61 | ~0.45 |
| Weight XGBoost | ~0.78 | ~0.53 | ~0.62 | ~0.46 |
| **Tuned SMOTE XGBoost** | **~0.78** | **~0.55** | **~0.65** | **~0.47** |
| Tuned Weight XGBoost | ~0.78 | ~0.54 | ~0.63 | ~0.47 |

Key findings:
- **PAY_0** (September payment status) and **AVG_PAY_STATUS** are the strongest predictors across all tree models
- **SMOTE and class-weight models** significantly improve recall (catching more defaults) at a modest precision cost
- **Lowering the threshold from 0.5 → 0.3** corrects for the class imbalance bias in baseline models
- **Tuned XGBoost** achieves the best overall F1 on the test set

## 🔧 Troubleshooting

### "Models could not be loaded — sklearn version mismatch"

The `.pkl` files were saved with a different scikit-learn version than the one installed. Fix:

```bash
pip install -r requirements.txt   # installs pinned scikit-learn==1.6.1
python model_train.py             # retrain with the current version
```

### "File exceeds 100 MB" on git push

Large Random Forest `.pkl` files must be tracked by Git LFS before staging. See the [Git LFS Setup](#-git-lfs-setup-for-large-model-files) section above.

### Feature Importance not showing

Feature importances are saved to `models/results_summary.json` during training. If this file is missing or was generated by an older version of `model_train.py`, re-run:

```bash
python model_train.py
```

### Widget labels appear dim / invisible

This is a CSS specificity issue with Streamlit's `var(--text)` variable. Ensure you are using the latest `app.py` which includes the explicit widget label overrides:

```css
[data-testid="stWidgetLabel"],
[data-testid="stWidgetLabel"] p { color: #E6EDF3 !important; }
```

### Port already in use

```bash
streamlit run app.py --server.port 8502
```
